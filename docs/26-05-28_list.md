# Rocky Linux 9 기준 UNIX 보안 점검 항목 검토 목록

작성일: 2026-05-28  
대상 문서: `docs/260423_서버보안 점검가이드(Unix).pdf`  
대상 스크립트: `linux_vuln_check.sh`  
전제: 현재 스크립트는 Rocky Linux 9/RHEL 9 환경을 기준으로 작성되어 있으며, PDF는 RHEL/CentOS 7 계열 기준 또는 과거 UNIX 관행이 일부 포함되어 있음.

## 1. RHEL/Rocky Linux 9 기준으로 그대로 추가하기 부적합하거나 대체가 필요한 항목

| PDF 항목 | 판단 | 사유 | 권장 방향 |
|---|---|---|---|
| UNIX01 사용자 계정에 0~99 UID/GID 값 할당 계정 확인 | 그대로 적용 비권장 | `0~99` 고정 기준은 구형 UNIX 계정 범위 기준에 가깝고, RHEL/Rocky 9에서는 `/etc/login.defs`의 `UID_MIN`, `GID_MIN` 기준으로 판단하는 것이 적절함. | 일반 사용자 UID/GID가 `UID_MIN/GID_MIN` 미만인지 확인하는 방식으로 대체 |
| UNIX07 네트워크 서비스 데몬 권한 755 초과 여부 설정 확인 | 대부분 부적합 | RHEL/Rocky 9는 systemd 기반이며 `/etc/rc.d/init.d`는 레거시 호환 영역임. | init.d 디렉터리가 존재할 때만 조건부 점검하고, 기본적으로 systemd unit/socket 권한 점검으로 대체 |
| UNIX18 inetd/xinetd가 불필요한 경우 비활성화 | 그대로 적용 부적합 | RHEL 9 계열에서는 xinetd가 기본 운영 모델이 아니며 systemd socket/service가 대체함. | xinetd 직접 점검보다는 systemd socket/service 활성 상태 점검으로 대체 |
| UNIX19 지정된 IP 외 주소에서 FTP/Telnet 접속 이력 점검 중 TCP Wrapper 기준 | TCP Wrapper 기준 부적합 | RHEL 8/9 계열에서는 TCP Wrappers가 제거되어 `/etc/hosts.allow`, `/etc/hosts.deny` 기반 판단은 부정확함. | firewalld/nftables/서비스 자체 ACL 및 로그 분석 기준으로 대체 |
| UNIX22 최신 버전 SSH 사용 | 업스트림 버전 숫자 기준 비권장 | RHEL 계열은 보안 패치를 백포트하므로 OpenSSH 업스트림 버전만으로 취약 여부를 판단하면 오탐 가능성이 있음. | `dnf updateinfo`, RHSA/RLSA 보안 업데이트 적용 여부 또는 배포판 패키지 보안 상태 기준으로 점검 |
| UNIX26 SSH 기본 포트 변경 및 Telnet/FTP 금지 중 “SSH 22번 포트 사용=취약” 기준 | 일반 기준으로 비권장 | SSH 기본 포트 22 사용 자체를 RHEL/Rocky 9 보안 취약점으로 단정하기 어려움. | SSH root 로그인 차단, 인증 방식, 접근 IP 제한, 방화벽 정책 점검을 우선하고 포트 변경은 조직 정책 항목으로 분리 |
| UNIX16 Web/WAS Default Page, Test Page 제거 | OS 기본 진단에는 조건부 | RHEL/Rocky 9와 충돌하지는 않지만 OS 취약점이라기보다 Apache/Tomcat/WAS 서비스 진단 영역임. | 웹 서버/WAS 설치 시 별도 서비스 모듈에서 조건부 점검 |

## 2. RHEL/Rocky Linux 9에서도 유효하며 추가 또는 보강을 권장하는 항목

| PDF 항목 | 현재 상태 | 추가/보강 권장 내용 |
|---|---:|---|
| UNIX01/UNIX02 계정 UID/GID 정책 | 부분 구현 | root 외 UID 0 점검은 있으나 GID 0 점검과 일반 사용자 UID/GID 범위 점검이 부족함. `UID_MIN/GID_MIN` 기준 미만 일반 계정, root 외 UID 0, root 외 GID 0 계정을 함께 점검 권장. |
| UNIX03 디폴트 시스템 계정 제거 | 부분 구현 | 현재 일부 계정만 확인함. `adm`, `sync`, `shutdown`, `halt`, `operator`, `squid`, `nuucp` 등 PDF 예시 계정과 Rocky 9 환경의 불필요 시스템 계정을 로그인 쉘/잠금 상태 기준으로 확장 권장. |
| UNIX04 로그인 실패 횟수 제한 | 보강 필요 | 현재 `deny == 5`만 양호로 볼 가능성이 있음. PDF 취지상 `deny <= 5`이면 양호로 보는 것이 적절함. `faillock.conf` 및 authselect/PAM 적용 상태까지 함께 확인 권장. |
| UNIX08 안전한 비밀번호 설정 | 보강 필요 | 현재 최소 길이 기준이 8 이상인 부분이 있음. PDF 기준 반영 시 `minlen >= 9`, 대문자/소문자/숫자/특수문자 요건, `PASS_MAX_DAYS <= 90`, `PASS_MIN_DAYS >= 1`을 통합 점검 권장. |
| UNIX11 root 권한상승, root 직접 접근, sh 권한 계정 패스워드 주기 관리 | 부분 구현 | SSH root 접속 제한과 su 제한은 일부 있으나 `sudoers` 및 `/etc/sudoers.d/*`의 `NOPASSWD` 허용 여부 점검이 부족함. 로그인 가능한 shell 계정의 패스워드 주기 정책도 함께 확인 권장. |
| UNIX15 익명 FTP 사용 여부 | 부분 구현 | `vsftpd.conf` 중심 점검만으로는 부족함. `/etc/passwd`의 `ftp`, `anonymous` 계정 존재 여부와 proftpd/pure-ftpd 설정까지 확인 권장. |
| UNIX17 자동 FTP 로그인을 허용하는 `.netrc` 파일 제거 | 미구현 | 사용자 홈 전체에서 `.netrc` 파일을 검색하고 존재 시 취약 처리하는 점검 추가 권장. |
| UNIX19 지정 IP 외 FTP/Telnet 접속 이력 점검 | 미구현/부분 구현 | 방화벽 또는 TCP Wrapper 존재 여부가 아니라 `/var/log/secure`, `last`, `lastlog`, FTP 로그 등을 기반으로 비인가 IP 접속 이력 분석 점검 추가 권장. 단, TCP Wrapper 기준은 RHEL/Rocky 9에 맞지 않으므로 제외. |
| UNIX20 Cron 파일 소유자 및 권한 설정 | 부분 구현 | 현재 `/etc/crontab` 중심임. `/etc/cron.allow`, `/etc/cron.deny`, `/etc/cron.d/*`, `/var/spool/cron`, `/usr/bin/crontab` 소유자/권한 점검 추가 권장. |
| UNIX21 디폴트 SNMP Community String 변경 | 부분 구현 | `community` 키워드 중심 점검은 net-snmp의 일반 설정인 `rocommunity public`, `rwcommunity private`를 놓칠 수 있음. `rocommunity`, `rwcommunity`, `com2sec` 등 실제 문법 기준으로 확장 권장. |
| UNIX23 IP 포워딩 비활성화 점검 | 미구현 | 일반 서버 기준 `net.ipv4.ip_forward=0` 런타임 및 `/etc/sysctl.conf`, `/etc/sysctl.d/*.conf` 영구 설정 점검 추가 권장. 라우터/게이트웨이 역할 서버는 예외 처리 필요. |
| UNIX24 패킷 재전송/SYN Flooding 대응 설정 | 미구현 | PDF 조치 기준인 `net.ipv4.tcp_syncookies=1` 런타임 및 영구 설정 점검 추가 권장. |
| UNIX25 Session Timeout 설정 | 부분 구현 | 현재 Shell `TMOUT` 일부만 확인함. `TMOUT <= 900`, `readonly TMOUT`, `export TMOUT`, `/etc/profile.d/*.sh` 설정 확인 추가 권장. SSH `ClientAliveInterval`, `ClientAliveCountMax`는 조직 정책에 따라 별도 점검으로 분리 가능. |
| UNIX26 Telnet/FTP 금지 | 부분 구현 | Telnet active 점검은 있으나 FTP는 enabled 중심이라 실행 중 프로세스나 LISTEN 포트를 놓칠 수 있음. systemd service/socket, 프로세스, 포트 상태를 함께 확인 권장. SSH 포트 변경은 필수 취약 기준보다 조직 정책 항목으로 분리 권장. |

## 3. 우선순위 제안

### 3.1 즉시 보강 우선순위 높음

- UNIX04 로그인 실패 횟수 제한
- UNIX08 비밀번호 정책
- UNIX11 `sudoers` NOPASSWD 및 root 권한상승 통제
- UNIX17 `.netrc` 파일 제거
- UNIX20 Cron 권한 범위 확장
- UNIX21 SNMP community string 점검 개선
- UNIX23 IP forwarding 비활성화
- UNIX24 TCP SYN cookies 설정
- UNIX25 Session Timeout 설정 보강

### 3.2 서비스 설치 시 조건부 보강

- UNIX15 익명 FTP
- UNIX16 Web/WAS Default Page, Test Page
- UNIX19 FTP/Telnet 접속 이력 분석
- UNIX26 Telnet/FTP 금지 및 원격관리 정책

### 3.3 RHEL/Rocky 9 방식으로 대체 권장

- UNIX01/UNIX02 계정 UID/GID 범위 점검
- UNIX07 init.d 권한 점검
- UNIX18 inetd/xinetd 비활성화
- UNIX22 최신 SSH 버전 점검

## 4. 현재 스크립트 근거 요약

- 현재 스크립트는 최신 매핑 흐름으로 `latest_check_u01`부터 `latest_check_u67`까지 실행함.
- 비밀번호 최소 길이 점검은 일부 로직에서 `minlen >= 8` 기준을 사용함.
- 로그인 실패 횟수 제한은 `deny == 5`처럼 고정값 중심으로 판단하는 부분이 있음.
- FTP 비활성화 점검은 `systemctl is-enabled` 중심이라 실행 중 상태나 포트 기반 탐지가 부족할 수 있음.
- SSH 접근제어는 `AllowUsers` 또는 `AllowGroups` 설정 중심이며, SSH 포트 변경 여부는 별도 점검하지 않음.
- SNMP community string 점검은 `community` 키워드 중심이라 `rocommunity`, `rwcommunity` 기반 설정 탐지 보강이 필요함.
