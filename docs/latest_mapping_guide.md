## MySQL/MariaDB: 기존 MX 체계 → 최신 D 체계

MariaDB는 MySQL과 호환되는 대상으로 간주하여 별도 코드로 구분하지 않습니다.

| 최신 코드 | 최신 항목 | 기존 대응 | 처리 |
|---|---|---|---|
| D-01 | 기본 계정의 비밀번호, 정책 등을 변경하여 사용 | MX-03 일부 | 재매핑 |
| D-02 | 데이터베이스의 불필요 계정을 제거하거나, 잠금설정 후 사용 | MX-02, MX-07, MX-08 | 통합 |
| D-03 | 비밀번호 사용 기간 및 복잡도를 기관의 정책에 맞도록 설정 | MX-04 | 재매핑 |
| D-04 | 데이터베이스 관리자 권한을 꼭 필요한 계정 및 그룹에 대해서만 허용 | MX-05 일부 | 재매핑 |
| D-06 | DB 사용자 계정을 개별적으로 부여하여 사용 | 없음 | 신규 |
| D-07 | root 권한으로 서비스 구동 제한 | 없음 | 신규 |
| D-08 | 안전한 암호화 알고리즘 사용 | 없음 | 신규 |
| D-10 | 원격에서 DB 서버로의 접속 제한 | MX-01, MX-06, MX-15, MX-16 | 통합 |
| D-11 | DBA 이외의 인가되지 않은 사용자가 시스템 테이블에 접근할 수 없도록 설정 | MX-05 일부 | 신규/부분 |
| D-14 | DB 주요 설정 파일 접근 권한 설정 | MX-09, MX-10 일부 | 통합 |
| D-21 | 인가되지 않은 GRANT OPTION 사용 제한 | 없음 | 신규 |
| D-25 | 주기적 보안 패치 및 벤더 권고 사항 적용 | MX-14 | 재매핑 |
